# F110 — Digital Island Apps

> AI-generated sandboxed micro-apps built via chat, saved as Interactives, embeddable on any site page. Price calculators, quiz widgets, booking forms — generated by conversation, previewed live, persisted in CMS.

## Problem

Creating interactive widgets (calculators, forms, quizzes, interactive maps) requires a developer. The Interactives Engine (F39) lets you manage and embed HTML components, but creating them still requires writing code in the admin's code editor or uploading HTML files. Non-technical users can't create interactive content.

The chat interface (F107) can already generate content, manage documents, and search media — but it can't create visual, interactive artifacts. When a user says "make me a price calculator", the AI should be able to generate it, show a live preview, and save it to the CMS — all within the conversation.

## Solution

Extend the chat interface with **artifact cards** — live-preview containers that render AI-generated HTML in a sandboxed iframe. The user can interact with the preview, request changes via follow-up messages, and save the result as a CMS Interactive (F39). Saved apps can be embedded on any site page using the existing Interactive block type.

Security: artifacts run in `sandbox="allow-scripts"` iframes with `srcdoc` — no network access, no cookie access, no parent DOM access. Complete isolation.

## Technical Design

### 1. Artifact Card Component

```typescript
// packages/cms-admin/src/components/chat/artifact-card.tsx

interface ArtifactCardProps {
  html: string;          // Full HTML document (self-contained)
  title: string;         // E.g. "Price Calculator"
  onSave?: () => void;   // Save to CMS Interactives
  onDownload?: () => void;
}
```

Layout:
```
┌─────────────────────────────────────────┐
│  🏝️ Price Calculator    [Preview] [Code]│
├─────────────────────────────────────────┤
│                                         │
│   ┌─────────────────────────────────┐   │
│   │  Sandboxed iframe with srcdoc   │   │
│   │  (live interactive preview)     │   │
│   │                                 │   │
│   └─────────────────────────────────┘   │
│                                         │
├─────────────────────────────────────────┤
│  [💾 Save to CMS]  [⬇ Download]        │
└─────────────────────────────────────────┘
```

- **Preview tab**: sandboxed iframe (`sandbox="allow-scripts"`, `srcdoc={html}`)
- **Code tab**: syntax-highlighted HTML source with copy button
- **Save to CMS**: calls `createInteractive` API → saved in Interactives library
- **Download**: browser download as `.html` file

### 2. Chat Tool: `generate_interactive`

```typescript
// Added to packages/cms-admin/src/lib/chat/tools.ts

{
  name: "generate_interactive",
  description: "Generate an interactive HTML component (calculator, form, quiz, chart, widget). Returns complete self-contained HTML.",
  input_schema: {
    type: "object",
    properties: {
      title: { type: "string", description: "Name for the interactive" },
      description: { type: "string", description: "What it should do" },
      style: { type: "string", description: "Visual style: 'minimal', 'branded', 'dark'" },
    },
    required: ["title", "description"],
  },
}
```

The tool calls Claude with a specialized system prompt for HTML generation:
- Self-contained HTML (inline CSS + JS, no external dependencies)
- Responsive design
- Dark mode support (reads `prefers-color-scheme`)
- No external network requests
- Accessible (ARIA labels, keyboard navigation)

The tool returns `__ARTIFACT__` prefix (like `__INLINE_FORM__`) which the SSE stream sends as an `artifact` event.

### 3. SSE Event Flow

```
User: "Make me a price calculator for cleaning services"
  → AI calls generate_interactive tool
  → Tool generates HTML via Claude
  → Returns __ARTIFACT__{title, html}
  → API sends: event: artifact, data: {title, html}
  → Chat UI renders ArtifactCard with live preview
  → User: "Add a discount field for repeat customers"
  → AI modifies HTML, sends new artifact
  → Updated preview replaces previous
```

### 4. Save to CMS

```typescript
// On "Save to CMS" click:
const response = await fetch("/api/media/interactives", {
  method: "POST",
  body: formData, // file: new Blob([html], {type: 'text/html'}), filename
});
// Returns InteractiveMeta with id
// User sees: "Saved! You can embed this on any page."
```

Uses the existing `MediaAdapter.createInteractive()` method.

### 5. Sandbox Security Model

```html
<iframe
  sandbox="allow-scripts"
  srcdoc={html}
  style="width:100%;height:400px;border:none;"
/>
```

**What's allowed:**
- JavaScript execution (allow-scripts)
- CSS animations, transitions
- Form elements, user interaction
- localStorage within sandbox origin

**What's blocked:**
- No `allow-same-origin` → can't access CMS cookies, API, or DOM
- No `allow-top-navigation` → can't redirect the page
- No `allow-popups` → can't open new windows
- No network requests to CMS server (srcdoc = opaque origin)
- No access to parent window via `window.parent`

### 6. postMessage API (for advanced apps)

Future extension: Digital Island Apps can communicate with the page via a controlled `postMessage` bridge:

```typescript
// Inside the artifact:
window.parent.postMessage({ type: "island:ready", id: "price-calc" }, "*");

// On the site page (loader script):
window.addEventListener("message", (e) => {
  if (e.data.type === "island:getTheme") {
    iframe.contentWindow.postMessage({
      type: "island:theme",
      colors: { primary: "#F7BB2E", bg: "#0D0D0D" }
    }, "*");
  }
});
```

This allows theme inheritance and data passing without compromising security.

## Relationship to Existing Features

- **F39 (Interactives Engine)** — F110 generates content FOR F39. Uses its storage and embedding. Doesn't change F39.
- **F58 (Interactive Islands)** — F58 is about Preact micro-apps compiled to tiny bundles for native rendering. F110 is about AI-generated HTML apps in sandboxed iframes. Different rendering model, complementary.
- **F107 (Chat with Your Site)** — F110 is a natural extension of F107, adding artifact generation to the chat's capabilities.
