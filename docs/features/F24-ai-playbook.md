# F24 — AI Playbook / Site Builder Guide

> Documentation and tooling that enables a blank AI session to build a complete site.

## Problem

When a developer starts a new AI coding session (Claude Code, Cursor, etc.), the AI has no context about @webhouse/cms. It cannot scaffold a site, configure collections, or generate content without extensive manual guidance.

## Solution

A comprehensive "playbook" system: CLAUDE.md files bundled in npm packages, project-level CLAUDE.md in scaffolded sites, MCP server for AI content access, pre-approved permissions in `.claude/settings.json`, and a `start.sh` one-command kickoff.

## Current State (In Progress)

Key files already shipped:

- `packages/cms/CLAUDE.md` — 822-line reference (complete CMS documentation for AI sessions)
- `packages/create-cms/` — scaffolds new sites with CLAUDE.md and `.claude/settings.json`
- `packages/cms-mcp-server/` — MCP server for AI content access

### What Exists

1. **CLAUDE.md in npm package** — Automatically available when `@webhouse/cms` is installed
2. **Project CLAUDE.md** — Generated by `npm create @webhouse/cms` with site-specific context
3. **`.claude/settings.json`** — Pre-approved tool permissions (file edit, bash commands)
4. **MCP server** — AI can query content via MCP tools

### What's Missing

1. **`start.sh` script** — One-command that starts dev server + opens AI session
2. **Playbook collection** — Curated task recipes (e.g., "add a blog", "set up i18n", "deploy to Vercel")
3. **AI-optimized error messages** — When something fails, error messages should suggest the AI command to fix it
4. **Interactive site builder prompt** — A starter prompt that walks through site setup decisions

## Technical Design

### start.sh

```bash
#!/bin/bash
# packages/create-cms/template/start.sh
# 1. Install dependencies if needed
# 2. Start CMS dev server in background
# 3. Open Claude Code with project context
# 4. Print "ready" message with available commands
```

### Playbook Recipes

```typescript
// packages/cms/src/template/playbooks/
// Each playbook is a markdown file with step-by-step AI instructions

export interface Playbook {
  id: string;
  title: string;
  description: string;
  prerequisites: string[];
  steps: string[];           // markdown instructions
  estimatedTime: string;
}
```

Playbooks:
- `add-blog.md` — Add a blog collection with posts, categories, authors
- `add-i18n.md` — Set up multi-language content
- `deploy-vercel.md` — Deploy to Vercel
- `add-podcast.md` — Set up podcast with episodes
- `customize-theme.md` — Configure brand colors and styles

### AI Error Messages

```typescript
// packages/cms/src/utils/errors.ts

export class CmsError extends Error {
  /** AI-friendly suggestion for fixing this error */
  aiHint: string;

  constructor(message: string, aiHint: string) {
    super(message);
    this.aiHint = aiHint;
  }
}

// Example:
throw new CmsError(
  'Collection "posts" not found in cms.config.ts',
  'Add a "posts" collection to cms.config.ts. Example:\n```\ndefineCollection({ name: "posts", fields: [...] })\n```'
);
```
