## Original Problem

The storage layer supports `locale` and `translationOf` fields, and `cms.config.ts` accepts `defaultLocale` and `locales`. But **nothing else is locale-aware**:

1. **AI generation is language-blind** — SEO, rewrite, proofread, agents, chat tools all call Claude without passing document locale. Character limits are hardcoded to English. Brand voice is single-language.
2. **Build pipeline outputs single-locale** — sitemap has no hreflang, routing has no locale prefixes, JSON-LD has no `inLanguage`, dates are hardcoded `en-US`.
3. **Admin UI has no translation workflow** — no locale filter, no language badge, no side-by-side editor, no settings to configure languages.
4. **No automation** — editor must manually create translations. No bulk translate, no auto-translate on publish, no stale translation detection.

This must be fixed NOW before the codebase grows larger. Every new feature that generates content will need locale hooks.

## Solution

Six phases, each independently shippable:

1. **Foundation** — locale settings UI, `getLocale()` helper, wire locale into all AI calls
2. **AI locale-awareness** — every prompt gets document locale, character limits adapt, brand voice auto-translates
3. **Translation management** — translation panel, side-by-side editor, API endpoints, stale detection
4. **Auto-translation** — on-publish hook auto-translates to all configured locales, bulk translate site
5. **Build pipeline** — locale URL prefixes, hreflang in sitemap, `inLanguage` in JSON-LD, locale-aware date formatting
6. **Chat + UI** — chat tools for translation, locale filter in collection list, UI localization

/** Get the locale for a document, falling back to site default */
export async function getDocLocale(doc: { locale?: string }): Promise<string> {

### 2.1 Locale-aware system prompt builder

**New file:** `packages/cms-admin/src/lib/ai/locale-prompt.ts`

```typescript
/**
 * Build locale instructions for any AI prompt.
 * Called by every AI route that generates content.
 */
export function buildLocaleInstruction(locale: string): string {
  const langName = LOCALE_LABELS[locale] ?? locale;
  return `LANGUAGE: Write ALL output in ${langName} (${locale}). This is non-negotiable — every word of generated content must be in ${langName}.`;
}

/**
 * Character limits vary by language (German/Finnish words are longer).
 * These are approximate optimal ranges for SERP display.
 */
export function getSeoLimits(locale: string): { titleMin: number; titleMax: number; descMin: number; descMax: number } {
  const compact = ["ja", "zh", "ko"]; // CJK: fewer chars needed
  const verbose = ["de", "fi", "nl"]; // compound words: need more chars
  if (compact.includes(locale)) return { titleMin: 15, titleMax: 30, descMin: 60, descMax: 80 };
  if (verbose.includes(locale)) return { titleMin: 35, titleMax: 65, descMin: 130, descMax: 165 };
  return { titleMin: 30, titleMax: 60, descMin: 120, descMax: 155 }; // default (en, da, fr, es, etc.)
}
```

### 2.3 Brand Voice per locale

```typescript
export async function getBrandVoiceForLocale(locale: string): Promise<BrandVoice | null> {
  const defaultVoice = await readBrandVoice();
  if (!defaultVoice) return null;
  if (defaultVoice.language === locale) return defaultVoice;
  // Check cache
  const cached = await readCachedBrandVoice(locale);
  if (cached) return cached;
  // Auto-translate (expensive — only done once per locale)
  const translated = await translateBrandVoice(defaultVoice, locale);
  await cacheBrandVoice(locale, translated);
  return translated;
}
```
