# cms-builder — F144 Phase 1

Ephemeral Builder VM image for cms-admin's Dynamic Site Build
Orchestrator (F144). Lives at `ghcr.io/webhousecode/cms-builder:latest`.

## What it does

When a user clicks rocket on an SSR site (Next.js, Bun/Hono) in
cms-admin, the orchestrator:

1. Tars the site source from `sites/<id>/source/`
2. Generates a per-framework Dockerfile
3. Spawns a `shared-cpu-4x@4096MB` Fly Machine in `arn` region using
   THIS image
4. The machine runs `entrypoint.sh` which:
   - Extracts source
   - Builds the image with `buildah` (rootless — no Docker daemon)
   - Pushes to `ghcr.io/webhousecode/<site-id>:<sha>`
   - POSTs status callbacks to cms-admin throughout
   - Auto-destroys (Fly's `auto_destroy=true`)
5. cms-admin receives the success callback + runs
   `flyctl deploy --image <ghcr-url> --app <target>`

## Why buildah/skopeo (not Docker-in-Docker)

Fly Machines don't expose a privileged Docker socket by default.
DinD requires `--privileged` which has security implications.
Buildah + Skopeo are rootless, daemonless equivalents that produce
OCI images compatible with anything that pulls them (Fly, Docker,
podman, k8s).

## Build + push

```bash
docker build -t ghcr.io/webhousecode/cms-builder:latest .
docker push ghcr.io/webhousecode/cms-builder:latest
```

The image only needs to be rebuilt when:
- Node base image LTS bumps
- pnpm / bun versions need updating
- We add a new tool to the entrypoint pipeline

It is NOT site-specific. One image serves all sites.

## Inputs (from cms-admin via Fly Machines `files:` + `env:`)

Files:
- `/build/source.tar.gz` — site source (no node_modules, .next, .git)
- `/build/Dockerfile` — framework template generated by cms-admin

Env:
- `SITE_ID` — for image tagging
- `SHA` — image tag (git commit or content hash)
- `TARGET_APP` — Fly app to deploy to (informational; cms-admin does
  the actual flyctl deploy after success)
- `REGISTRY_TOKEN` — short-lived GHCR push token
- `CALLBACK_URL` — cms-admin endpoint for status updates
- `CALLBACK_TOKEN` — bearer for the callback POST

## Output

- Image at `ghcr.io/webhousecode/<SITE_ID>:<SHA>` (versioned)
- Image at `ghcr.io/webhousecode/<SITE_ID>:latest`
- Status callbacks: `init`, `source-extract`, `image-build`,
  `image-push`, `done` (or `failed` with last 4 KB of build log)

## Cost

`shared-cpu-4x@4096MB` ≈ $0.11/hour. A 5-min Next.js build = $0.009.
100 deploys/month ≈ $1.

## Lifecycle

| Phase | Wall time | Notes |
|---|---|---|
| Machine spawn | 5-15 sec | Fly cold-start |
| Source extract | <1 sec | tar of <10 MB typical |
| Image build | 2-7 min | Next.js install + build |
| Image push | 30-90 sec | First push: full layers; subsequent: incremental |
| Auto-destroy | <5 sec | Fly tears down |

Total: 4-9 min per build. cms-admin's flyctl deploy on top adds 30-60 sec
for rolling restart + smoke-test.
